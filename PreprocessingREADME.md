## Pipeline

### Step 1 - Preprocessing

Parses PAN 2012 XML, applies the rule engine to every predator message, computes conversation-level risk prefix tokens, and saves train/val/test splits as JSON.

```bash
python preprocess.py
```

**Config block at the top of `preprocess.py`:**

| Variable | Default | Description |
|---|---|---|
| `TRAIN_XML` | `data/train/pan12-...xml` | Training corpus XML |
| `TRAIN_PRED` | `data/train/pan12-...txt` | Predator author IDs |
| `TEST_XML` | `data/test/pan12-...xml` | Test corpus XML |
| `TEST_GT1` | `data/test/pan12-...-problem1.txt` | Test ground truth |
| `OUTPUT` | `pan12_dataset` | Output directory |
| `VAL_RATIO` | `0.1` | Fraction of train used for validation |
| `SEED` | `42` | Random seed for split |

**Output format** (one record per conversation):

```json
{
  "conversation_id": "affc2df...",
  "text": "[RISK:HIGH] [CATS:isolation,image_solicitation] [ESC:1] [ARC:1] [DOM:0] [PRED] [RULE:image_solicitation:3] send me a pic [SEP] [USER] idk",
  "label": 1
}
```

**Prefix tokens injected into every conversation:**

| Token | Meaning |
|---|---|
| `[RISK:HIGH/MEDIUM/LOW]` | Conversation-level risk level |
| `[CATS:cat1,cat2,...]` | Active rule categories |
| `[ESC:1]` | Severity escalates in second half |
| `[ARC:1]` | Trust-building → solicitation stage arc detected |
| `[DOM:1]` | Predator sends >60% of messages |

**Risk score formula:**

```
score = Σ weight(cat) × min(count, 3)
      + Σ co-occurrence bonuses (21 dangerous pairs)
      + escalation bonus (up to +4.5)
      + stage arc bonus (up to +3.5)
      + dominance bonus (+1.5)
      + consecutive message bonus (+0.5 per run of 3)
```

Thresholds: HIGH ≥ 6.0, MEDIUM ≥ 2.5.

### Step 2 - Training

Fine-tunes `microsoft/deberta-v3-base` (183M parameters) on the preprocessed dataset.

```bash
python train.py
```

**Config block at the top of `train.py`:**

| Variable | Default | Description |
|---|---|---|
| `DATASET_PATH` | `pan12_dataset` | Output of preprocess.py |
| `MODEL_NAME` | `microsoft/deberta-v3-base` | Base model |
| `OUTPUT_DIR` | `runs/pan12` | Where to save checkpoints |
| `EPOCHS` | `5` | Max training epochs |
| `LR` | `2e-5` | Learning rate |
| `BATCH_SIZE` | `16` | Per-device batch size |
| `GRAD_ACCUM` | `2` | Gradient accumulation steps (effective batch = 32) |
| `PATIENCE` | `2` | Early stopping patience |

**Training details:**

- Loss: weighted cross-entropy with `w_pos = 3.0 × (N_neg / N_pos) ≈ 29.4`
- Metric: F₂ score (recall weighted 2× over precision)
- Decision threshold: calibrated on val set each epoch via grid search, not fixed at 0.5
- Optimizer: AdamW (β₁=0.9, β₂=0.999, weight decay=0.01)
- Scheduler: linear warmup (10%) + linear decay

**Outputs:**

- `runs/pan12/best/` - best checkpoint (HuggingFace format, loadable with `AutoModelForSequenceClassification`)
- `runs/pan12/best_threshold.json` - optimal decision threshold
- `runs/pan12/test_results.json` - precision, recall, F2, TP, FP, FN on test set

### Step 4 - Context model

Sliding window (N=30) per-contact tracker combining ML scores and rule hits into three-level alerts.

```python
from context import ContextModel
from rules import RuleEngine

engine = RuleEngine()
ctx    = ContextModel()

alert = ctx.update(
    text="send me a pic",
    sender="external",       # "external" or "child"
    ml_prob=0.91,
    rule_matches=engine.match("send me a pic"),
)
if alert:
    print(alert)             # [CRITICAL] Explicit grooming indicator | cats=[image_solicitation]
```

**Alert levels:**

| Level | Trigger |
|---|---|
| Critical | Any severity-3 rule match in current message |
| High | ≥ 3 messages with ml_prob ≥ 0.80, OR window risk score ≥ 6.0 |
| Medium | ≥ 5 risky messages, OR window risk score ≥ 2.5 |

The window risk score uses the same formula as preprocessing, computed over the last 30 messages.

| Language | English |
| Source | Perverted-Justice.com sting logs |

- `training-corpus-predators.txt` - author-level labels for training (one ID per line)

## Known limitations

- All training data is English; cross-lingual performance degrades 15–25 F1 points for related languages (French, Spanish, German) with no fine-tuning.
- PAN 2012 is sourced from IRC chat (2000–2010); modern messaging register differs. Slang injection and back-translation augmentation are not yet implemented.
- The rule engine fires false positives on some benign contexts (e.g. `"don't tell your parents about the surprise party"` → sev=2). Negative lookaheads reduce but do not eliminate this.
- iOS accessibility restrictions make full message-level capture unavailable without MDM profiles. Android is fully supported via Accessibility Service.
- Adversarial evasion (coded language, pacing, platform migration) is not modelled. The rule engine is lexical; a determined adversary can evade it while the ML component provides some robustness.
